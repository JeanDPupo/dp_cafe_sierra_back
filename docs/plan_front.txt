Plan de Implementación: Frontend Plataforma de Comercialización de Café
Context
Proyecto: Prototipo de Plataforma web para la comercialización directa de Café entre Caficultores del Corregimiento de Sacramento y Consumidores Finales - Universidad del Magdalena.

Objetivo: Crear un frontend completamente funcional y usable que demuestre el flujo completo de la plataforma con rol switching entre Productor y Consumidor.

Tech Stack:

React 19.2.6 (ya configurado en /mi-frontend/)
Ubicación: c:\desarrollo\UNIMAG_projects\PrototipoWebComercializaciónCaféSacramento\mi-frontend\
Insights Clave de las Entrevistas:

MVP real necesita solo 3 funciones: productor publica → consumidor encuentra → contactan
PWA crítico para zonas con conectividad intermitente
Simplicidad de interfaz es fundamental para usuarios rurales
Propuesta de valor debe ser concreta
Acompañamiento humano/contexto es esencial
Funcionalidades Comprometidas (MVP)
F1: Registro de Usuarios - Diferenciar roles (Productor/Consumidor)
F2: Publicación de Productos - Productor publica café con foto, precio, cantidad, ubicación
F3: Catálogo - Consumidor visualiza cafés disponibles con filtros básicos
F4: Contacto vía WhatsApp - Botón directo para contactar productor
F5: Perfil del Productor - Vista de quién produce y desde dónde
Approach Recomendado
Fase 1: Estructura Base y Navegación
